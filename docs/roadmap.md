# Roadmap

## Suggested Order

1. Keep demo mode stable and make sure every contract has tests.
2. Implement the real feature extraction layer and lock its versioning strategy.
3. Replace the semantic and lexical generators with human-owned strategies.
4. Add a real composer that reasons about cross-group interference.
5. Introduce a stronger solver backend and ambiguity evaluator.
6. Replace the baseline scorer with human-owned ranking formulas.
7. Expand the frontend from demo inspection to richer editorial tooling.

## MVP to Stronger System

### MVP

- Demo-mode end-to-end generation
- Typed schemas and API
- Static and generated puzzle responses
- Clear human-owned seams

### Stronger Generator

- Curated feature store
- Better candidate diversity
- Real rejection filtering
- Offline evaluation corpus

### Production-Oriented System

- Solver ensembles
- Batch evaluation jobs
- Metrics dashboards
- Editorial review workflow
- Persistent puzzle storage and audit logs
